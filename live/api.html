Quick auth

Three lines and you're authenticated.

Mint a key in Settings → API, drop it in an env var, and call any endpoint. Bearer-token auth over HTTPS, exactly like every other modern API. OAuth 2.0 for user-scoped flows when you're building for someone else.

Quick auth · cURL example 
# 1 · Set your key (mint at Settings → API)
export TT_API_KEY="tt_live_..."

# 2 · Authenticate every request with a Bearer token
curl https://api.tradetron.tech/v1/strategies \
  -H "Authorization: Bearer $TT_API_KEY"

# → 200 OK · { "strategies": [...] }
Endpoints

A small, sharp surface.

Base · https://api.tradetron.tech/v1
POST /strategies Build a strategy programmatically. Same schema the no-code canvas uses — multi-leg options, conditions, sets.
POST /strategies/{id}/backtest Run a backtest over years of historical data. Worst-case slippage, walk-forward, realism flags returned in the response.
POST /strategies/{id}/deploy Deploy or pause a strategy against a connected broker. Paper, Live-Offline or Live-Auto.
POST /strategies/{id}/audit AI Strategy Audit. Same audit that gates the Go-Live button in the UI — invoke it from your own deploy pipeline before you ship a strategy live.
POST /signals Fire an entry / exit signal into a deployed strategy. The executor routes the order into your user's broker.
GET /strategies/{id}/positions Read open positions, fills and live PnL — broker-confirmed, the same numbers the dashboard shows.
POST /webhooks Register outbound webhooks for fills, broker disconnects, drawdown breaches and Universal Exit events.

Full OpenAPI reference, schemas and per-endpoint examples live at api.tradetron.tech/docs.

Auth model

Two ways in. Pick the one that fits.

Build for yourself with a static API key. Build for someone else — and have their funds stay in their broker, not yours — with OAuth.

For your own account
API key (Bearer token)

Mint in Settings → API. Scope to read-only or full trading. Rotate or revoke any time. Best for personal scripts, TradingView webhooks, in-house dashboards.

For your users' accounts
OAuth 2.0 (user-scoped)

Standard authorization-code flow. Your user logs in to Tradetron, grants scopes, and you get a token bound to their account. Their funds never leave their broker — Tradetron never custodies a paisa.
